# Deploying Parallax (Render + Cloudflare + Google sign-in)

> ⚠️ **Why not Cloudflare Pages/Workers?** Parallax is a Python server that spawns
> CLI subprocesses (maigret, sherlock, gosearch, phoneinfoga…) and runs multi-minute
> scans. Workers/Pages have no Python runtime, no subprocess, no long-running server —
> so the app **cannot** run there. It needs a container host. Cloudflare then sits in
> front for your domain + TLS + CDN.

The app runs **open** locally (no login). Google sign-in turns on automatically once
`GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` are present — i.e. only in the cloud.

---

## 1. Google OAuth credentials (free)

1. <https://console.cloud.google.com> → create a project.
2. **APIs & Services → OAuth consent screen** → *External* → fill app name/email →
   **Publish app** (moves it out of "testing" so **any** Google account can sign in;
   the `openid email profile` scopes are non-sensitive, no verification needed).
3. **APIs & Services → Credentials → Create Credentials → OAuth client ID** →
   *Web application*.
   - Authorized redirect URI: `https://YOUR-DOMAIN/auth/callback`
     (use the Render URL first, then your Cloudflare domain — you can list both).
4. Copy the **Client ID** and **Client secret**.

---

## 2. Deploy to Render (GitHub auto-deploy)

1. <https://render.com> → **New → Blueprint** → connect this GitHub repo.
   Render reads [`render.yaml`](render.yaml) and builds the [`Dockerfile`](Dockerfile).
2. Set the environment variables it prompts for:
   | Var | Value |
   |-----|-------|
   | `GOOGLE_CLIENT_ID` | from step 1 |
   | `GOOGLE_CLIENT_SECRET` | from step 1 |
   | `OAUTH_REDIRECT_URL` | `https://YOUR-DOMAIN/auth/callback` |
   | `SESSION_SECRET` | auto-generated by Render |
   | `ALLOWED_EMAILS` | *(optional)* comma-separated allow-list; blank = any Google account |
3. Deploy. Every push to `main` auto-redeploys.

First build takes a few minutes (compiles gosearch from Go, fetches phoneinfoga,
installs the engines).

---

## 3. Put Cloudflare in front

1. In Cloudflare DNS for your domain, add a **CNAME** →
   `parallax.onrender.com` (your Render URL), **Proxied** (orange cloud).
2. In Render → **Settings → Custom Domains** → add `YOUR-DOMAIN`.
3. Update `OAUTH_REDIRECT_URL` (Render env) **and** the Google redirect URI to the
   Cloudflare domain. Redeploy.

Cloudflare now terminates TLS and proxies to Render. `--proxy-headers` in the
Dockerfile makes the app build correct `https://` OAuth URLs behind the proxy.

---

## Free-tier caveats

- Render Free = **512 MB RAM / 0.1 CPU**, and **spins down when idle** (first request
  after a nap is slow to wake).
- **maigret** scans all ~3000 sites — on 0.1 CPU it is slow and can strain 512 MB.
  If it OOMs, cap it (edit `app/engines/maigret.py`: drop `-a`, add `--top-sites 500`)
  or move to a paid instance.
- The **Stop** button + per-job subprocess kill keep runaway scans bounded.

## Security reminder

A public OSINT scanner can be abused and may hit target-site rate limits from your
server IP. Keep it behind Google sign-in (default here), optionally restrict with
`ALLOWED_EMAILS`, and only scan subjects you're authorized to investigate.
